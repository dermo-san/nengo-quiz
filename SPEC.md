- **実データ（塾プリントの120問）はこのリポジトリに含めない・渡されない。** 開発は本SPECの仕様に従って自分で生成するダミーデータ（§6.3）で行う。
- ビルド工程なし。vanilla HTML/CSS/JS の静的サイト。npm・フレームワーク・バンドラ禁止。
- 触ってよいのは `/Users/shuhei/nengo-quiz/` 配下のみ。
- `.gitignore` に `questions.real.json` と `*.real.json` を必ず入れる（実データ混入防止）。

## 1. 目的・背景

- 中学受験する小6の子が、塾の「重要年代（標準編）120問」（8列×15問）を暗記するためのアプリ。
- 塾で4回のテストがあり、**最低8割**（未達は居残り）、目標9割以上。
- テスト日程と範囲（左から2列ずつ）:
  - 第1回 2026-09-01(火): 列1-2（問1〜30）
  - 第2回 2026-09-08(火): 列3-4（問31〜60）
  - 第3回 2026-09-15(火): 列5-6（問61〜90）
  - 第4回 2026-09-29(火): 列7-8（問91〜120）
- **2列=30問=塾テスト1回分=アプリ1セッション**。この対応が設計の軸。
- 利用: iPad Safari（ホーム画面追加=PWA）。利用者は子ども1人。アカウント・サーバ・課金なし。

## 2. 機能要件

### 2.1 出題モード
1. **回別モード（メイン）**: 第1回〜第4回を選択 → その範囲30問（順番どおり／シャッフルを選択可）
2. **苦手優先モード**: 全範囲または回別から、間違いの多い問題を優先して30問（§2.6）
3. **全範囲ランダム**: 120問から重複なしで30問
4. **間違いだけ再挑戦**: 結果画面から、直前セッションの誤答のみ再出題（30問未満で可）

### 2.2 解答フロー
- 出来事テキストを大きく表示。進捗「n/30」を常時表示。
- **画面内カスタムテンキー**で入力: 0〜9・1文字消す・クリア・解答。iOSソフトキーボードは一切使わない（`input`要素にフォーカスさせない設計にする）。
- **入力は1〜4桁**（2桁の年号が存在する。例: 57年）。5桁目は入力不可。先頭0は無視して整数として扱う。空入力のとき解答ボタンは無効。
- 判定: 入力値と `year` の整数完全一致。
- **紀元前は v1 非対応**（データに存在しない前提。実データ化で発見された場合のみ「BC」トグルを後付けする——実装不要、拡張余地としてUIを詰めすぎないこと）。
- 判定後: ○/×を大きく表示。正しい「出来事＋年号」を必ず提示。×のときは自分の答えも併記。`goro` が非空なら表示（実データでは当面すべて空）。
- 「次へ」で次問。戻る・スキップはなし。

### 2.3 結果画面
- 正解率（%）を大きく表示し、**80%ライン（合格）と90%ライン（目標）**をゲージ上に明示。
- 間違えた問題の一覧（出来事・正しい年号・自分の答え）。
- ボタン: 「間違いだけ再挑戦」「もう一回30問」「ホームへ」。

### 2.4 モチベーション機能
- **テストカウントダウン**: ホームに「第n回テストまであとm日」を4回分表示（§1の日付をハードコードでよい。過ぎた回は「終了」表示）。
- **合格メーター**: 回ごとに、その範囲の直近セッション正解率（または直近3回平均）をゲージ表示。80%・90%ラインを明示。
- **コンボ**: セッション中の連続正解数を表示。自己ベスト更新時に演出。
- **紙吹雪演出**: セッション90%以上でCSS/canvasの紙吹雪＋称賛メッセージ。100%は特別バージョン。外部ライブラリ不可、自前実装。
- **スタンプカレンダー**: 実施した日にスタンプが付く月表示カレンダー。連続日数（ストリーク）表示。
- **マスター制度**: 各問題「3連続正解でマスター」。回ごとのマスター数（n/30）を表示。
- 文言は小6向けに短くポジティブに。

### 2.5 記録・統計
- 問題別: 出題回数・正解数・現在の連続正解数・マスターフラグ。
- セッションログ: 日時・モード・対象回・出題数・正解数。
- **エクスポート/インポート**: 学習記録＋問題データをJSONテキストで書き出し（テキストエリア表示＋コピー）/読み込み。

### 2.6 苦手優先の選定ロジック（シンプルでよい）
- 各問題に重み = 1 + 誤答数×2 + （直近結果が誤答なら+3） − マスター済みなら重み最小（0.2）。
- 重み付きランダムで重複なしに30問抽出。凝ったSRSは不要。

### 2.7 問題データの読み込み
- 優先順位: localStorage保存済みデータ > ファイル読み込み。
- 設定画面に「問題ファイルを読み込む」(`<input type="file" accept=".json">`) と「サンプルデータで試す」（同梱 `questions.sample.json` をfetch）。
- 読み込み時にスキーマ検証（id 1〜120重複なし・year整数・round 1〜4）。不正ならエラー内容を表示して反映しない。
- 実データは公開リポジトリ・公開URLに置かない（塾プリントの編集著作物性への配慮）。ユーザーがiPadの「ファイル」経由で読み込む運用。

## 3. 非機能要件

- iPad Safari（縦持ち優先、横も崩れない）。PWA: `manifest.json`＋Service Worker（キャッシュファースト）でホーム画面追加・オフライン動作。
- 全処理クライアント内・即時応答。外部通信なし（フォント・CDN含め外部リソース禁止）。
- localStorage保存。ホーム画面PWAはSafariの7日消去(ITP)対象外だが、保険としてエクスポート機能（§2.5）。

## 5. 画面構成

1. **ホーム**: カウントダウン×4／合格メーター×4／モード選択／カレンダー・設定への導線

## 6. データ設計

### 6.1 questions.json スキーマ
```json
{
  "version": "2026-07-08",
  "title": "重要年代（標準編）120問",
  "questions": [
    { "id": 1, "col": 1, "row": 1, "round": 1,
      "event": "（出来事のテキスト）", "year": 645, "goro": "" }
  ]
}
```
- `id`: 1〜120（列順の通し番号）。`col`: 1〜8、`row`: 1〜15、`round`: col 1-2→1, 3-4→2, 5-6→3, 7-8→4。
- `year`: 西暦の正の整数（1〜4桁）。`goro`: 任意・空文字可。

### 6.2 localStorage キー
- `nengo.questions` / `nengo.stats` / `nengo.sessions` / `nengo.settings`（構造は実装に任せるが、エクスポートはこの4つを1JSONに束ねる）

### 6.3 ダミーデータ（Codexが生成して同梱）
- 120問。`event` は答えが分かるテスト用文言にする: 例 `"ダミー出来事（列3・7番目）〔こたえ: 794〕"`。
- `year` は**2桁・3桁・4桁を必ず混在**させる（2桁を最低3問、3桁を最低5問含める）。同一年号の重複が数問あってもよい（実データにもありうる）。
- `goro` は全問空文字（表示機能の動作確認用に2〜3問だけ入れてよい）。

- [ ] 回別モードが正しい30問を出す。シャッフル・順番どおりが機能する

- [ ] 間違いだけ再挑戦が誤答のみ出題する
- [ ] リロード・ブラウザ再起動後も成績・データが残る
